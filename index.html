Hackathon Problem 8 — Detecting Fake Banking APKs

Goal: Build an automated system that flags malicious banking APKs using static/dynamic analysis and intelligence feeds.

  • Static: Manifest, permissions, signing cert, strings/URLs.
  • Dynamic (heuristics): overlay creation, SMS/OTP read, a11y abuse.
  • ML: combine features into a risk score (0–100).
  • Intel: enrich domains/IPs, WHOIS, ASN, blocklists.

FAQ

Does the demo upload my APK?

No. The prototype does not upload files. Analysis and risk scoring are simulated in your browser for UX demonstration.

How would a real system work?

A backend would run a sandbox (e.g., Android emulator) and static tooling, enrich IOCs via TI feeds, and return a signed JSON report.

Is there a CLI?

Planned: a simple CLI wrapping the same scoring to integrate with CI and takedown workflows.

Is it open-source?

The demo UI can be open-sourced; model weights and TI connectors may remain private due to abuse risk.
